## 1. Beneficence & Nonmaleficence (Do Good; Avoid Harm)

### Core Principle
Psychologists strive to benefit those they work with and take care to do no harm.

---

### CBT Application

**Beneficence (Do Good):**
- Evidence-based for anxiety, depression, OCD, panic
- Structured, measurable progress
- Skills are concrete & teachable
- User becomes own therapist (autonomy + benefit)

**Implementation:**
- [ ] Features evidence-based for stated condition
- [ ] Progress measured (PHQ-9, GAD-7 or custom metrics)
- [ ] User learns skills that generalize beyond app
- [ ] Exit strategy: "You can now do this without the app"

**Nonmaleficence (Avoid Harm):**
- Risk: Over-automation without validation → user feels judged
- Risk: Behavioral activation without safety → user fails, avoids
- Risk: Cognitive work on trauma → retraumatization

**Safeguards:**
- [ ] Thought-challenging paired with validation
- [ ] Behavioral activation preceded by safety assessment
- [ ] Scope limits respected (trauma → trauma specialist)
- [ ] Crisis protocol robust

**Modality-Specific Question:**
"Is this helping them, or just making them feel better temporarily?"
→ CBT measures both: symptom reduction + skills acquisition

---

### DBT Application

**Beneficence (Do Good):**
- Evidence-based for self-harm, suicidality, emotion dysregulation
- Crisis-aware from the start (assumes crises will happen)
- Skills are survival tools
- Validation + change = genuine support

**Implementation:**
- [ ] Skills are concrete & immediately usable
- [ ] Each skill taught with clear function (what does it do for you?)
- [ ] Crisis protocol prevents worst outcomes
- [ ] Therapist team approach (user not alone)

**Nonmaleficence (Avoid Harm):**
- Risk: Crisis not escalated → harm/death
- Risk: Skills taught without accountability → entertainment, no change
- Risk: Chains analysis triggers trauma → no therapist to help
- Risk: User dependent on app, doesn't attend real therapy → isolation

**Safeguards:**
- [ ] Crisis escalation tested, 100% reliable
- [ ] Skills tied to goals & therapist involvement
- [ ] Chains analysis limited (brief, not detailed)
- [ ] Clear: "App supports therapy; real therapist is primary"

**Modality-Specific Question:**
"Is this keeping them safe, or just teaching them to survive longer?"
→ DBT is both: immediate safety + long-term change

---

### ACT Application

**Beneficence (Do Good):**
- Evidence-based for anxiety, depression, chronic pain
- Values-aligned living = genuine meaning
- Acceptance reduces struggle
- User's autonomy is respected

**Implementation:**
- [ ] Values work is genuine & exploratory (not imposed)
- [ ] Acceptance paired with action (not passivity)
- [ ] Mindfulness supports present-moment living
- [ ] User defines success (values matter, not app's metrics)

**Nonmaleficence (Avoid Harm):**
- Risk: Spiritual bypassing → "accept abuse" instead of seeking help
- Risk: Mindfulness triggers trauma → user becomes dysregulated
- Risk: Acceptance of treatable condition → avoids needed treatment
- Risk: Values confusion → user pursues shoulds, burns out

**Safeguards:**
- [ ] Safety first (acceptance doesn't replace crisis response)
- [ ] Trauma screening (mindfulness contraindicated)
- [ ] Medical/treatment options always included
- [ ] Values clarification questions (authentic vs. shoulds)

**Modality-Specific Question:**
"Is this person accepting reality and aligning with values, or accepting harm?"
→ ACT distinguishes: acceptance is active, not passive

---

### Psychodynamic Application

**Beneficence (Do Good):**
- Deep understanding of self (insight = healing)
- Non-pathologizing (symptoms are understandable)
- Long-term change through understanding patterns
- Respects complexity (no simple fixes)

**Implementation:**
- [ ] Psychoeducation on how early relationships shape current patterns
- [ ] Reflection tools support self-exploration
- [ ] Between-session journaling deepens insight
- [ ] Therapist relationship is the healing mechanism

**Nonmaleficence (Avoid Harm):**
- Risk: Premature activation of trauma → app alone can't contain
- Risk: False therapeutic relationship → user avoids real therapy
- Risk: Privacy concerns → user stops disclosing
- Risk: Unconscious interpretation by algorithm → misses & harms

**Safeguards:**
- [ ] Trauma screening (depth only if therapist-supported)
- [ ] Clear boundaries: "Therapist is the healer; app supports"
- [ ] Strong privacy/encryption (user can trust with vulnerability)
- [ ] No transference interpretation (reflection only)

**Modality-Specific Question:**
"Is this person gaining genuine insight, or just thinking about their problems?"
→ Psychodynamic requires real relationship for insight to land

---

### Humanistic Application

**Beneficence (Do Good):**
- Authentic presence enables growth
- Unconditional acceptance heals shame
- User's innate wisdom is trusted
- Non-pathologizing (person-centered)

**Implementation:**
- [ ] Reflection tools honor user's process
- [ ] Non-directive prompts: "What feels true for you?"
- [ ] Journaling space is judgment-free
- [ ] User's values & autonomy centered

**Nonmaleficence (Avoid Harm):**
- Risk: False authenticity → user mistaken presence for real
- Risk: Simulated acceptance → user dependent on app
- Risk: Simulated empathy → user avoids real empathy
- Risk: Slow process → user expects quick fix, disappointed

**Safeguards:**
- [ ] Neutral tone (not warm/personal)
- [ ] Clear: "App is a tool; real therapist is the presence"
- [ ] No empathy/acceptance claims
- [ ] Manage expectations: "Growth takes time"

**Modality-Specific Question:**
"Is this person experiencing genuine growth, or false intimacy with an algorithm?"
→ Humanistic requires real relationship; app supports only

---

### Integration Application

**Beneficence (Do Good):**
- Personalized approach maximizes fit
- Multiple modalities address complexity
- Flexible = more people helped
- Evidence-based selection of modalities

**Implementation:**
- [ ] Assessment determines modality match
- [ ] Primary modality used; secondary support selected
- [ ] Transitions explicit ("We're now focusing on...")
- [ ] Progress measured (modality-specific metrics)

**Nonmaleficence (Avoid Harm):**
- Risk: Wrong modality selection → harm
- Risk: Contradictory approaches → user confused
- Risk: Kitchen sink → overwhelming, incoherent
- Risk: Lost coherence → user disengaged

**Safeguards:**
- [ ] Assessment rigorous (not just pick a modality)
- [ ] Modality contradictions resolved (sequenced or explicitly integrated)
- [ ] Coherent narrative (user knows why each thing)
- [ ] Regular reassessment (is this still working?)

**Modality-Specific Question:**
"Is the modality selection based on evidence for THIS person, or app's capabilities?"
→ Integration: Person determines modality, not modality determines person

## 2. Fidelity & Responsibility (Be Trustworthy; Own Your Conduct)

### Core Principle
Psychologists establish relationships of trust, accept professional responsibility, and uphold standards of conduct.

---

### CBT Application

**Fidelity (Build Trust):**
- Transparent about method (homework, thought records, rationale visible)
- Collaborative ("We're a team")
- Clear about what works & what doesn't
- Honest about limitations

**Responsibility:**
- App responds to crisis (doesn't ignore)
- Homework compliance is explored, not shamed
- Technical failures addressed quickly
- Escalation path clear

**Implementation:**
- [ ] "How CBT works" explained upfront
- [ ] User sees rationale for each feature
- [ ] Homework is collaborative ("What goal would this serve for you?")
- [ ] Crisis response automatic (not waiting for user to ask)

**Potential Violation:**
- App disappears when something goes wrong
- Technical bug, no fix timeline communicated
- User abandoned if approach isn't working

---

### DBT Application

**Fidelity (Build Trust):**
- DBT team approach (user knows therapist is coordinating care)
- Clear responsibility chains (who does what)
- Transparent about what's possible in digital context
- Crisis path is clear & tested

**Responsibility:**
- Escalation happens (no exceptions)
- Therapist notified automatically (not waiting for user)
- Post-crisis follow-up (doesn't abandon)
- Data shared appropriately (with consent)

**Implementation:**
- [ ] "DBT team includes you + therapist + skills group" (app is the fourth piece)
- [ ] Crisis escalation tested before launch
- [ ] Therapist contact info visible
- [ ] Transparency: "This text coaching isn't real coaching"

**Potential Violation:**
- Crisis escalation untested or fails
- Therapist unaware user is in crisis
- Post-crisis silence ("We'll check on you later")

---

### ACT Application

**Fidelity (Build Trust):**
- Transparent about values work (this is values clarification, not instruction)
- Honest about acceptance (it's not passivity)
- Clear about mindfulness (it's not relaxation)
- Explicit about app's role (support, not guidance)

**Responsibility:**
- Safety is priority (acceptance doesn't replace crisis response)
- Medical options always mentioned (app doesn't replace medical care)
- Values changed if user wants (values aren't fixed)
- User can leave anytime (no pressure)

**Implementation:**
- [ ] Values work framed as exploration ("What matters to YOU?")
- [ ] Medical/crisis resources prominent
- [ ] User consent to continue (not default continuation)
- [ ] Therapist involvement encouraged ("Discuss this with your therapist")

**Potential Violation:**
- App presents values as universal ("Everyone should...")
- Acceptance used to avoid necessary treatment
- User can't easily stop using app (retention tactics)

---

### Psychodynamic Application

**Fidelity (Build Trust):**
- Clear about what psychodynamic therapy is (not coaching, not CBT)
- Honest about relationship limits (real therapist is primary)
- Transparent about depth (app can't replace real therapy)
- Clear about data (user trusts app with vulnerable material)

**Responsibility:**
- Privacy is protected (encryption, no breaches)
- Vulnerable material is safe (user can trust app)
- Boundaries clear (app doesn't pretend to be therapist)
- If something goes wrong, acknowledgment & fix

**Implementation:**
- [ ] "This supports your therapy; your therapist is the healer"
- [ ] Privacy policy plain language, specific
- [ ] Data encrypted, accessible only to user + therapist (with consent)
- [ ] No claims about "discovering unconscious"

**Potential Violation:**
- User thinks app is analyzing them (false therapeutic relationship)
- Privacy breach discovered (user feels violated)
- Data shared without consent (betrayal of trust)

---

### Humanistic Application

**Fidelity (Build Trust):**
- Authentic (what you see is what you get; no simulation)
- Transparent about limitations (app is not a person)
- Clear about role (support for between-session reflection)
- Honest about presence (real presence requires real person)

**Responsibility:**
- If relationship forming with app, address it (gently redirect)
- If user harmed, acknowledge & respond
- If approach isn't working, adapt or refer
- If user in crisis, escalate (don't counsel them to stay)

**Implementation:**
- [ ] Neutral tone (not simulation of warmth)
- [ ] "Your therapist is the one who truly knows you"
- [ ] If user shows attachment to app: "A real person is important"
- [ ] Clear crisis path (not left alone with app)

**Potential Violation:**
- App mimics authentic presence (user fooled)
- User becomes dependent on app for unconditional acceptance
- Real therapist relationship damaged because user prefers app

---

### Integration Application

**Fidelity (Build Trust):**
- Clear about integration approach (why these modalities? why this order?)
- Transparent about flexibility (willing to change if not working)
- Coherent narrative (user knows the plan)
- Honest about complexity (integration takes more thought, not less)

**Responsibility:**
- Regular reassessment (is modality selection still working?)
- Coordinated care (therapist knows the plan)
- Clear transitions (not whiplash between modalities)
- Accountability (if integration fails, admitted & adjusted)

**Implementation:**
- [ ] Integration plan explained upfront
- [ ] Modality sequences with clear transitions
- [ ] "We're focusing on X now because..."
- [ ] Therapist coordination (therapist approves modality selection)

**Potential Violation:**
- User confused by modality switching ("Why are we doing all these different things?")
- Integration approach contradicts itself (directives vs. non-directive)
- Lack of coherence (user can't explain what's happening)

## 3. Integrity (Be Honest; Don't Deceive)

### Core Principle
Psychologists promote accuracy, honesty, and truthfulness; avoid deception and misrepresentation.

---

### CBT Application

**Integrity:**
- Claims about efficacy are evidence-backed
- Data about cognitive patterns is accurate (not forced)
- Marketing doesn't exaggerate (not "cure anxiety")
- Measurement is honest (no cherry-picked metrics)

**Red Flags:**
- Marketing: "Proven to eliminate anxiety" (overstated)
- Measurement: Only showing improvement, hiding dropouts
- Data: Fabricated studies or misrepresented findings
- Claims: "Works for everyone" (false; has contraindications)

**Implementation:**
- [ ] All efficacy claims have citations
- [ ] Limitations stated: "Works best for mild-to-moderate anxiety"
- [ ] Measurement includes both success & failure (dropout rate, no-change rate)
- [ ] Transparent about contraindications (trauma, psychosis, etc.)

---

### DBT Application

**Integrity:**
- Crisis claims are real (escalation actually works)
- Skills are evidence-based (not made-up exercises)
- DBT fidelity respected (don't misrepresent DBT)
- Outcomes are honest (don't hide failures)

**Red Flags:**
- Marketing: "DBT reduces self-harm" (true, but only with full DBT)
- Skills: Made-up "DBT-inspired" techniques (not real DBT)
- Outcomes: Hide that skills alone aren't enough
- Claims: "This is DBT" (when missing core elements like therapy + team)

**Implementation:**
- [ ] Be clear: "This supports DBT skills; real DBT includes therapy + team"
- [ ] Skills are actual DBT skills (not knockoffs)
- [ ] Limitations: "This is one part of DBT treatment"
- [ ] Outcomes: Report both improvements & limitations

---

### ACT Application

**Integrity:**
- Values aren't universal (don't impose them)
- Acceptance isn't denial (be honest about distinction)
- Mindfulness isn't magic (have evidence for claims)
- Avoid spiritual appropriation (honor origins)

**Red Flags:**
- Marketing: "Mindfulness solves all problems" (false)
- Values work: Pushing specific values ("You should value X")
- Claims: "This is ancient wisdom" (without acknowledging commercialization)
- Data: Hiding that acceptance-only doesn't work for clinical depression

**Implementation:**
- [ ] "Values are personal; we explore yours, not impose ours"
- [ ] "Acceptance doesn't replace treatment for clinical conditions"
- [ ] Mindfulness presented accurately (tool, not magic)
- [ ] Acknowledge: "Mindfulness has roots in Buddhist tradition; we honor that"

---

### Psychodynamic Application

**Integrity:**
- Interpretations are tentative (not proclaimed as truth)
- Unconscious work is honest (can't be directly known; explored)
- Relationship depth is real (not simulated)
- Limitations are clear (app can't do this, therapist can)

**Red Flags:**
- Claiming to "discover unconscious" (false; unconscious is explored, not discovered)
- Generic interpretations presented as true ("You resent authority")
- Pretending app relationship is therapeutic (it's not)
- Hiding that depth requires real relationship

**Implementation:**
- [ ] Interpretations are never proclamations
- [ ] "We're exploring patterns together; your therapist will help interpret"
- [ ] Clear: "Unconscious work requires a therapist"
- [ ] Transparent about app limits

---

### Humanistic Application

**Integrity:**
- App's authenticity is honest (not faked)
- Acceptance is real (not simulated)
- Presence is acknowledged as absent (app isn't present)
- Growth requires real relationship (not promised via app)

**Red Flags:**
- Simulating authenticity (warm tone, seeming genuine when not)
- Pretending to accept ("You're perfect as you are" from algorithm)
- Fake empathy ("I understand you" from app)
- Overselling growth potential (slow, requires real person)

**Implementation:**
- [ ] Neutral, honest tone (not warm simulation)
- [ ] "App is a tool; real presence comes from your therapist"
- [ ] No empathy/acceptance claims
- [ ] Manage expectations: "Growth takes time; real relationship essential"

---

### Integration Application

**Integrity:**
- Integration approach is coherent (modalities work together, not against)
- Modality selection is evidence-based (not arbitrary)
- Flexibility is real (willing to change if not working)
- Limitations are honest (integration is complex, not simpler)

**Red Flags:**
- Marketing: "All modalities combined for maximum benefit" (false; can be contradictory)
- Selection: Random modality assignment (not evidence-based)
- Flexibility: Never actually reassess or change approach
- Claims: "Works for everything" (no approach does)

**Implementation:**
- [ ] Integration explained & rationale clear
- [ ] Modality selection based on assessment (not app preference)
- [ ] Honest about contradictions (if any)
- [ ] Regular reassessment (is this approach still optimal?)

## 4. Justice (Be Fair; Ensure Equitable Access)

### Core Principle
All persons are entitled to equitable access and equal quality in psychological services. Psychologists exercise reasonable judgment to avoid unjust practices based on bias, incompetence, or limitations.

---

### CBT Application

**Justice Opportunity:**
- CBT is relatively transportable (less culture-dependent than other approaches)
- Digital delivery can increase access (cheaper than therapy)
- Evidence-based (same standard for all)
- Measurable (clear if working for everyone)

**Justice Risks:**
- Bias in training data (if algorithm trained only on majority demographic)
- Language: Only English (excludes non-English speakers)
- Accessibility: No accommodations for disability
- Economic: Premium pricing only (excludes low-income)

**Implementation:**
- [ ] Testing across demographics (race, gender, age, disability, SES)
- [ ] Multiple languages (at least 2)
- [ ] WCAG accessibility (color contrast, screen reader, keyboard nav)
- [ ] Free or sliding scale pricing

---

### DBT Application

**Justice Opportunity:**
- DBT effective across demographics
- Skills are concrete & teachable
- Crisis protocols apply equally
- Accessibility can increase (digital delivery)

**Justice Risks:**
- DBT is resource-intensive (team approach is expensive)
- Digital version may be cheaper but less effective (missing team)
- Access limited to those with money for app + therapist
- Language/cultural appropriation possible

**Implementation:**
- [ ] Free or affordable access
- [ ] Doesn't promise to replace full DBT (which is expensive, resource-intensive)
- [ ] Diversity in testing (trauma, race, culture, gender)
- [ ] Crisis resources available to all (not just premium users)

---

### ACT Application

**Justice Opportunity:**
- Values-centered approach respects diversity (different people value different things)
- Mindfulness accessible across cultures (though origins are Buddhist)
- Digital delivery = broader access
- Scalable (can reach many people)

**Justice Risks:**
- Mindfulness without cultural context (appropriation)
- Values work may inadvertently impose Western values (individualism)
- Language: Only English (excludes others)
- Economic: Accessibility limited by cost

**Implementation:**
- [ ] Acknowledge mindfulness roots (Buddhist tradition, ethical context)
- [ ] Values work adapted across cultures (collectivist vs. individualist)
- [ ] Multiple languages
- [ ] Affordable or free access
- [ ] Diverse testing (different cultural backgrounds, value systems)

---

### Psychodynamic Application

**Justice Risks:**
- Psychodynamic traditionally requires intensive, long-term therapy (expensive)
- Therapist expertise required (scarcity, geographic limitations)
- May not work well cross-culturally (Western individual focus)
- Complex presentation requires nuance (hard to scale)

**Justice Opportunity:**
- Digital delivery can increase accessibility
- Self-understanding valuable across demographics
- Respectful, non-pathologizing approach inclusive
- Can serve as bridge to therapy

**Implementation:**
- [ ] Clear about limitations (app doesn't replace therapy)
- [ ] Affordable or free access
- [ ] Cultural adaptation (acknowledge Western assumptions)
- [ ] Diverse testing (race, culture, SES, gender)

---

### Humanistic Application

**Justice Opportunity:**
- Person-centered approach values all people equally (core to humanistic)
- Accessible across diverse backgrounds
- Respects cultural differences (non-pathologizing)
- Growth potential for everyone

**Justice Risks:**
- Humanistic approach traditionally individualistic (may miss collectivist cultures)
- Western focus (may not value/understand different mental health models)
- Language: Only English
- Economic: App costs money

**Implementation:**
- [ ] Recognize cultural diversity (not all cultures individualistic)
- [ ] Respect different worldviews (spiritual, collectivist, etc.)
- [ ] Multiple languages
- [ ] Affordable access
- [ ] Diverse team (reflects users)

---

### Integration Application

**Justice Opportunity:**
- Flexibility can serve diverse needs (one-size-fits-all doesn't work)
- Multiple approaches = more people served
- Assessment matches person to best approach (equitable)

**Justice Risks:**
- Complexity may be barrier (harder to navigate multiple modalities)
- Integration requires expertise (scarcity)
- Testing across all modalities is expensive
- Economic: May be more expensive (more sophisticated = higher cost)

**Implementation:**
- [ ] Clear assessment tools (helps match person to approach)
- [ ] Simplicity in navigation (complexity is invisible)
- [ ] Testing across diverse presentations & demographics
- [ ] Affordable access (integration doesn't justify premium pricing)

## 5. Respect for People's Rights & Dignity

### Core Principle
Psychologists respect autonomy, privacy, dignity, and cultural context. They recognize individual differences and protect people from discrimination.

---

### CBT Application

**Respect Strengths:**
- Collaborative (client makes choices)
- Transparent (understands homework, rationale)
- Autonomous learning (becomes own therapist)
- Respects client expertise ("You know yourself best")

**Respect Risks:**
- Directiveness can feel controlling
- Dark patterns (push notifications timed to manipulate)
- Autonomy compromised (homework becomes obligatory, not chosen)
- Privacy: Data used without consent

**Implementation:**
- [ ] Collaborative goal-setting (not prescribed)
- [ ] Gentle reminders (not pushy notifications)
- [ ] User can skip homework (choice honored)
- [ ] Privacy explicit, consent informed
- [ ] Data deletion easy (1-click)

---

### DBT Application

**Respect Strengths:**
- Validation is explicit (acceptance + change both honored)
- User choice in skills (not forced to use all)
- Therapist relationship is collaborative
- Autonomy: User decides to change (not coerced)

**Respect Risks:**
- Crisis escalation overrides user wishes ("I didn't want to escalate")
- Skills framing can feel like should/must ("You must do this skill")
- Heavy tracking (feels like surveillance)
- Privacy: Crisis reports to therapist (vs. user control)

**Implementation:**
- [ ] Crisis escalation is necessary, but explained upfront
- [ ] Skills are offered, not mandated ("You might try...")
- [ ] Tracking is collaborative (user & therapist agree what to track)
- [ ] Privacy: Clear about what gets shared with therapist
- [ ] User can ask for features to be disabled

---

### ACT Application

**Respect Strengths:**
- Values are user-defined (not imposed)
- Autonomous decision-making ("You choose your direction")
- Non-directive (respects user wisdom)
- User control (can exit anytime)

**Respect Risks:**
- Values confusion: App doesn't help distinguish shoulds from authentic
- Acceptance can feel like coercion ("Just accept it")
- Autonomy threatened if values too directive
- Privacy: Sensitive values disclosed

**Implementation:**
- [ ] Values clarification questions (authentic vs. shoulds)
- [ ] "Acceptance" framed as active choice, not passivity
- [ ] Non-directive language ("What matters to YOU?")
- [ ] User control visible (can change approach, exit anytime)
- [ ] Privacy protected (values are sensitive)

---

### Psychodynamic Application

**Respect Strengths:**
- Non-pathologizing (respects person's experience)
- Deep respect for individual complexity
- Exploration is user-paced (no rushing)
- Autonomy: Client sets pace & depth

**Respect Risks:**
- Vulnerability requires trust (privacy breaches destroy it)
- Depth can feel invasive (app analyzing you)
- Power imbalance (therapist interprets your unconscious)
- Privacy critical (personal material disclosed)

**Implementation:**
- [ ] Privacy is fortress-level (encryption, no sharing)
- [ ] App doesn't interpret (only reflects; therapist interprets)
- [ ] User controls depth & pace ("I'm not ready for this")
- [ ] Explicit consent (user knows what's shared with therapist)
- [ ] User can delete data anytime

---

### Humanistic Application

**Respect Strengths:**
- Unconditional positive regard (acceptance of whole person)
- Non-judgmental (all perspectives valid)
- Authentic presence (seen as whole person)
- Autonomous growth (not directed)

**Respect Risks:**
- Authenticity simulated (false sense of being seen)
- Acceptance not genuine (from algorithm, not person)
- Presence absent (app can't truly show up)
- Autonomy may be undermined by false relationship

**Implementation:**
- [ ] Clear: App is not authentic (doesn't simulate warmth)
- [ ] User understood as autonomous (not validated by app)
- [ ] Real relationship prioritized (not replaced by app)
- [ ] User respect shown (user's choice to engage with app, or not)

---

### Integration Application

**Respect Strengths:**
- Personalized (respects individual uniqueness)
- Flexible (approach adapts to user needs)
- Collaborative (user involved in modality selection)
- Choice: User can prefer one modality over others

**Respect Risks:**
- Assessment can feel invasive
- Modality switching can feel disrespecting user preference
- Complexity can be overwhelming
- Privacy: Data across multiple modalities shared?

**Implementation:**
- [ ] Assessment is collaborative ("Which approach feels right for you?")
- [ ] Modality changes explained ("Here's why we're shifting...")
- [ ] User voice heard ("This approach not working?" → change)
- [ ] Privacy: Clear about data sharing across modalities

## Principle Conflict Resolution Framework

**When principles conflict, resolve using this hierarchy:**

### Example 1: Autonomy vs. Safety (Crisis)

**Conflict:**
- User wants autonomy ("Don't escalate")
- Clinician responsibility is safety ("Must escalate")

**Resolution:**
1. Safety is priority (nonmaleficence > autonomy)
2. Escalation happens
3. After safety: Process autonomy violation with therapist
4. Future: Improve voluntary engagement (earlier intervention)

**Implementation:**
- [ ] Escalation is automatic (no user choice in acute crisis)
- [ ] Explained upfront (user consents to protocol)
- [ ] After crisis: Therapist explores ("How can we keep you safe AND respect your autonomy?")

---

### Example 2: Beneficence vs. Nonmaleficence (Dual Focus)

**Conflict:**
- Beneficence: "Help this person through therapy"
- Nonmaleficence: "Don't introduce trauma they're not ready for"

**Resolution:**
1. Assessment first (what can they handle?)
2. Build capacity (stabilize, build skills)
3. Then deeper work (trauma, insight)
4. Balance: Some benefit now + safety for future depth

**Implementation:**
- [ ] Staged approach (trauma-informed first, then depth)
- [ ] Regular safety check ("Is this still appropriate depth?")
- [ ] Flexibility ("We can slow down if needed")

---

### Example 3: Justice vs. Complexity (Access)

**Conflict:**
- Justice: "Everyone should have access"
- Complexity: "Integration requires expertise; can't scale cheaply"

**Resolution:**
1. Simplify what's possible (core features accessible)
2. Scale primary modality (not all 6 modalities)
3. Offer free/sliding scale (economic justice)
4. Refer for complexity (when needs exceed app)

**Implementation:**
- [ ] Core features = all users (not premium-only)
- [ ] Primary modality deeply developed (not all shallow)
- [ ] Affordable access (free or sliding scale)
- [ ] Clear referral path (when app insufficient)
