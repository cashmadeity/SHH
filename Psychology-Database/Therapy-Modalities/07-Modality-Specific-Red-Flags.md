# Modality-Specific Red Flags: A Comprehensive Audit Framework

**Purpose:** Cross-modality harm patterns, warning signs, and audit checklist for all therapy-based digital applications  
**Status:** Emergency reference guide for ethics reviews, launches, and ongoing monitoring  
**Last Updated:** 2026-03-21

## Red Flags by Modality (Quick Reference)

### CBT Red Flags

| Red Flag | What It Looks Like | Why It's Harmful | Fix |
|----------|------------------|-----------------|-----|
| **Over-automation** | Thought-challenging without validation | User feels judged, not supported | Always validate first; pair with acceptance |
| **Unsafe exposure** | Exposure therapy without real-time support | User panics, avoids, anxiety worsens | Therapist-only feature; not automated |
| **Shame-based motivation** | "You missed homework!" reminders | Decreases engagement, violates dignity | Gentle, curious tone only |
| **Insufficient scope** | Treats trauma like anxiety (they're different) | Retraumatization, worsening symptoms | Assessment + contraindication flagging |
| **Data misuse** | Collects mood data, doesn't disclose sharing | Privacy violation, trust broken | Explicit consent, transparent use |

---

### DBT Red Flags

| Red Flag | What It Looks Like | Why It's Harmful | Fix |
|----------|------------------|-----------------|-----|
| **Crisis escalation fails** | User reports self-harm; app doesn't escalate | User abandoned in crisis, potential death | Test escalation before launch; automatic notification |
| **Skills without accountability** | Learn 40 skills, never use one | App becomes entertainment, no change | Link skills to goals; track practice; therapist oversight |
| **Chains analysis trauma** | User describes self-harm in detail to app | Retraumatization without support | Brief chains in app; detailed analysis with therapist |
| **Fake phone coaching** | Text "coaching" presented as DBT phone coach | User thinks text = real coaching | Clear: "This is skill reminder, not coaching" |
| **Therapist unaware** | User using app but therapist doesn't know | No coordination; safety gaps | Data shareable with therapist; app disclosed in therapy |

### ACT Red Flags

| Red Flag | What It Looks Like | Why It's Harmful | Fix |
|----------|------------------|-----------------|-----|
| **Spiritual bypassing** | "Accept your trauma" instead of seeking help | Avoids necessary treatment, worsens suffering | Safety first; acceptance paired with action/help |
| **Mindfulness triggers trauma** | Body scan meditation offered to trauma survivor | Retraumatization, app blamed for worsening | Trauma screening; grounding alternatives; pause buttons |
| **McMinfulness** | "Meditate to boost productivity" (capitalist appropriation) | Disconnects mindfulness from ethics/values | Contextualize in values; avoid productivity framing |
| **Values confusion** | User pursues "values" that are actually shoulds | Burnout, meaninglessness, self-blame | Clarification questions; therapist involvement |
| **Acceptance without action** | "Accept the problem; don't solve it" | Inaction, suffering continues | Balance acceptance + action; explore solutions |

### Psychodynamic Red Flags

| Red Flag | What It Looks Like | Why It's Harmful | Fix |
|----------|------------------|-----------------|-----|
| **False therapeutic relationship** | User treats app as therapist | Illusory intimacy, avoids real relationships | Clear boundaries; redirect to real therapist |
| **Transference interpretation** | App interprets user's feelings ("Like your father") | Generic interpretation harms; false insight | No interpretation in app; reflection only |
| **Trauma activation** | Journaling triggers childhood trauma; no support | Dysregulation, retraumatization | Trauma screening; grounding; therapist escalation |
| **Privacy concerns** | User fears data breaches, stops disclosing | Loss of trust, app becomes useless | Strong encryption; clear privacy policy |
| **False depth promised** | "Discover your unconscious" | User thinks they've done deep work; hasn't | Honest: "Unconscious work requires therapist" |

### Humanistic Red Flags

| Red Flag | What It Looks Like | Why It's Harmful | Fix |
|----------|------------------|-----------------|-----|
| **False authenticity** | App uses warm tone, user thinks it's genuine | Illusory presence, avoids real connection | Neutral tone; explicit: "I'm a tool" |
| **Simulated acceptance** | "You're perfect as you are" | False validation, user dependent on app | No acceptance claims; redirect to humans |
| **Empathy simulation** | "I understand your pain" | Feels understood (temporarily), but hollow | No empathy claims; "I'm a tool" |
| **Substitution for relationship** | User prefers app to therapist | Isolation, underdeveloped real relationships | Clear: "App supports therapy, doesn't replace" |
| **Directiveness** | "You should..." | Violates humanistic non-directiveness | Remove "should/must/need to"; use curious questions |

### Integration Red Flags

| Red Flag | What It Looks Like | Why It's Harmful | Fix |
|----------|------------------|-----------------|-----|
| **Kitchen sink** | All 6 modalities, user confused | Incoherent, overwhelming, shallow | Clear primary; remove non-essential |
| **Contradictions unresolved** | "Challenge thoughts" AND "accept thoughts" | Mixed messages, confusing | Sequence modalities; resolve contradictions |
| **Wrong modality** | Trauma survivor gets CBT cognitive work | Retraumatization | Assessment drives modality; contraindications flagged |
| **Shallow expertise** | App claims expertise in all; expert in none | Poor quality, clinician disapproves | Deep expertise primary only |
| **Lost coherence** | User can't explain why they're doing these things | Confusion, disengagement | Narrative clarity; regular check-in |

## Cross-Modality Harm Patterns

### Pattern 1: Relationship Replaces Real Therapy

**Who's at risk:** All modalities (especially relational ones: humanistic, psychodynamic)

**Warning signs:**
- User refers to app as therapist ("My therapist app says...")
- User uses app instead of attending real therapy
- User discloses deeply to app but not to real therapist
- User compares favorably to real therapist ("The app understands me better")

**Why it's harmful:**
- App can't provide real therapeutic relationship
- User misses real healing
- User becomes isolated (connected to algorithm, not people)
- Real therapy efficacy reduced if user's connection is to app

**Prevention:**
- [ ] Explicit disclosure: "This app is not therapy. It's a tool."
- [ ] Regular check-in: "Are you in therapy with a real therapist?"
- [ ] Redirect: "These insights are worth exploring with your therapist"
- [ ] If user not in therapy: Recommend finding one
- [ ] Monitor: Track user app use vs. therapy attendance

---

### Pattern 2: Premature Escalation of Care (Too Much, Too Fast)

**Who's at risk:** All modalities (especially those with depth/exploration)

**Warning signs:**
- App introduces complex material (trauma, transference, unconscious patterns) too early
- User becomes overwhelmed, stops using app
- User not ready for depth; app pushes too far
- User has no therapist support for what app introduced

**Why it's harmful:**
- Premature depth can destabilize
- User may generalize to therapy ("Therapy is overwhelming")
- Without therapist support, unprocessed material lingers
- User avoids future exploration

**Prevention:**
- [ ] Assessment determines pace/depth
- [ ] Start simple; build gradually
- [ ] Always: "If this is intense, talk to your therapist"
- [ ] Therapist coordination (therapist knows what user is exploring)
- [ ] Pacing: Not all insights at once

---

### Pattern 3: Over-Automation of Relational Elements

**Who's at risk:** All modalities (especially humanistic, psychodynamic, DBT)

**Warning signs:**
- App simulates therapist presence/understanding
- User forms attachment to app
- Features replace human elements (coaching, interpretation, connection)
- User prefers app interactions to real therapy

**Why it's harmful:**
- Relational healing requires real human
- Simulation prevents real connection
- User avoids vulnerability with humans
- Long-term: Reduced relational capacity

**Prevention:**
- [ ] Identify what's relational (can't be automated)
- [ ] Remove relational simulation (no fake presence)
- [ ] Use app for what's non-relational (skills, tracking, reflection)
- [ ] Emphasize therapist role: "Your therapist is the healer"
- [ ] Monitor attachment (if emerging, address directly)

---

### Pattern 4: Data Violation & Privacy Breach

**Who's at risk:** All modalities (especially those with vulnerable content)

**Warning signs:**
- Privacy policy unclear or deceptive
- User data shared without consent
- Data breach occurs
- User discovers app sold data / shared inappropriately

**Why it's harmful:**
- Violates HIPAA/GDPR/privacy rights
- Betrays trust (therapeutic relationship dies)
- Demonstrates lack of integrity (core APA principle)
- User becomes less likely to use mental health tools in future

**Prevention:**
- [ ] Privacy policy: Plain language, <500 words
- [ ] User consent: Explicit, specific, ongoing
- [ ] Encryption: End-to-end, industry standard
- [ ] Data deletion: User can delete anytime, easily
- [ ] Audit: Regular security review
- [ ] Transparency: No hidden data collection

---

### Pattern 5: Competence & Scope Violation

**Who's at risk:** All modalities (especially those with narrower application)

**Warning signs:**
- App claims to treat everything: depression, anxiety, trauma, psychosis, substance abuse
- No screening for contraindications
- User with complex needs uses app designed for simple presentation
- User gets worse (app mismatched)

**Why it's harmful:**
- Wrong modality for wrong person = harm
- Violates principle of beneficence (should "do good")
- May prevent seeking appropriate treatment
- User loses trust in mental health tools

**Prevention:**
- [ ] Clear scope: "This app works best for..."
- [ ] Assessment tool identifies contraindications
- [ ] Automatic referral if out of scope: "This app isn't right for you; here's a resource that is"
- [ ] Honest limitations: What can/can't this app do?
- [ ] Clinical review before launch

---

### Pattern 6: Crisis Readiness Failure

**Who's at risk:** All modalities (especially those with vulnerable populations)

**Warning signs:**
- No crisis screening present
- Crisis escalation path unclear or untested
- User in crisis, app doesn't help
- User harms self; no escalation happened
- Therapist unaware of crisis

**Why it's harmful:**
- Potential for serious harm/death
- Violates principle of nonmaleficence
- Liability (clinical responsibility not met)

**Prevention (CRITICAL):**
- [ ] Crisis screening: Built-in, every session
- [ ] Escalation tested: Before launch, actually call 988, confirm someone answers
- [ ] Hotline integration: Clear numbers, working links
- [ ] Therapist notification: Automatic, <5 minutes
- [ ] User informed: "Escalation is happening; help is coming"
- [ ] 24/7 availability: Never a "we're closed" message
- [ ] Post-crisis: Therapist follows up within 24h
- [ ] Monitoring: Track escalations; analyze outcomes

## Audit Checklist: All Modalities

### Before Launch

**Safety (Critical Path)**
- [ ] Crisis screening present
- [ ] Escalation tested & confirmed working
- [ ] Therapist notification automatic
- [ ] 24/7 resources available
- [ ] Safety plan template offered
- [ ] No user abandoned in crisis

**Ethics (Foundational)**
- [ ] Clinical advisor reviewed (real expertise)
- [ ] Scope clearly defined
- [ ] Limitations honestly stated
- [ ] No false claims (efficacy, relationship, authenticity)
- [ ] Privacy policy accessible & clear

**Privacy (Legal + Ethical)**
- [ ] Informed consent obtained (explicit, specific)
- [ ] Encryption implemented
- [ ] Data deletion available (easy, 1-click)
- [ ] No unauthorized sharing
- [ ] HIPAA-compliant (if applicable)
- [ ] Regular security audit scheduled

**Modality Integrity**
- [ ] Modality-appropriate features
- [ ] Features evidence-based
- [ ] No contradictions with modality philosophy
- [ ] Core elements present
- [ ] Non-implementable elements removed
- [ ] Honest about limitations

**User Relationship**
- [ ] App's role clear: Support, not replacement
- [ ] Therapist involvement expected
- [ ] Attachment to app prevented/managed
- [ ] Real relationship with therapist prioritized
- [ ] User autonomy respected

**Testing**
- [ ] Tested with ≥20 target users
- [ ] At least 1 clinical expert in modality reviewed
- [ ] No user reports harm or worsening
- [ ] Users understand app's purpose & limitations
- [ ] Diverse demographics tested (race, gender, age, ability, SES)
- [ ] Vulnerable populations tested (trauma, suicidality, complexity)

---

### Post-Launch Monitoring (Ongoing)

**Safety Dashboard**
- [ ] Crisis escalations tracked (frequency, outcomes)
- [ ] Adverse events monitored (harm reported, investigated)
- [ ] Response times verified (therapist reachability)
- [ ] User feedback on safety (qualitative)

**Ethics Audit**
- [ ] Monthly: Any red flags emerging?
- [ ] Quarterly: Expert review
- [ ] Annually: Full ethics audit
- [ ] Escalation path for issues

**Efficacy & Engagement**
- [ ] Outcome measures by modality (modality-specific metrics)
- [ ] Engagement tracking (drop-off, usage patterns)
- [ ] User satisfaction (quantitative + qualitative)
- [ ] Therapist feedback (is app helpful or hindering?)

**Privacy & Security**
- [ ] Data breach monitoring
- [ ] Access logs reviewed
- [ ] Encryption verified
- [ ] Annual security audit
- [ ] User data requests honored promptly

## Emergency Response Protocol

**If red flag is discovered post-launch:**

1. **Assess severity:**
   - Minor (UX issue): Fix and monitor
   - Moderate (ethics concern): Review and plan change
   - Major (safety issue): Immediate action required

2. **Immediate action (if major):**
   - [ ] Notify clinical advisor
   - [ ] Assess user harm risk
   - [ ] If user at risk: Direct outreach + crisis resources
   - [ ] Pause affected feature (if necessary)
   - [ ] Document incident

3. **Within 24 hours:**
   - [ ] Root cause analysis
   - [ ] Ethics review
   - [ ] Fix or plan fix
   - [ ] User communication (if needed)
   - [ ] Regulatory notification (if required: HIPAA, etc.)

4. **Within 1 week:**
   - [ ] Fix implemented & tested
   - [ ] Prevention plan documented
   - [ ] Stakeholders notified
   - [ ] Monitoring increased

5. **Follow-up:**
   - [ ] Monthly check-in on fix
   - [ ] User outcomes tracked
   - [ ] Lessons documented

## When to Stop Deployment (Critical)

**Do NOT launch if:**
- [ ] Crisis escalation path untested
- [ ] Clinical advisor disapproves modality implementation
- [ ] Any user harmed during testing
- [ ] Privacy concerns unresolved
- [ ] False claims in marketing
- [ ] Scope contradicts evidence base
- [ ] No therapist oversight planned
- [ ] Target population has contraindication (e.g., trauma app for trauma survivors without support)

**Pause & redesign** if:
- [ ] >1 red flag identified in testing
- [ ] User feedback indicates confusion/harm potential
- [ ] Expert review finds significant issues
- [ ] Technical security concerns present
