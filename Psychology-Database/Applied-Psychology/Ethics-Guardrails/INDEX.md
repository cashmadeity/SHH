## Key Principles at a Glance

| Principle | Means | Red Flag | Best Practice |
|-----------|-------|----------|----------------|
| **Beneficence** | Do good | Addiction mechanics | Evidence-based features |
| **Fidelity** | Be trustworthy | Dark patterns | Clear disclosures |
| **Integrity** | Be honest | False claims | Transparent methodology |
| **Justice** | Be fair | One-size-fits-all | Diverse testing, equitable access |
| **Respect** | Honor autonomy | Dark patterns | Informed consent, user choice |

## Before You Launch: Critical Checklist

- [ ] **Red Flags:** All critical items passed ([Red-Flags/](Red-Flags/))
- [ ] **Legal:** Consent, privacy, data security compliant ([Legal-Compliance/](Legal-Compliance/))
- [ ] **Clinical:** Scope clear, crisis escalation, therapeutic alliance ([Best-Practices/](Best-Practices/))
- [ ] **Validation:** Level 1 & 2 passed; Level 4 recommended ([Validation-Framework.md](Validation-Framework.md))
- [ ] **Diversity:** Tested with 50+ users across demographics ([Validation-Framework.md#level-4](Validation-Framework.md))

**Do NOT launch if any critical item fails.**
