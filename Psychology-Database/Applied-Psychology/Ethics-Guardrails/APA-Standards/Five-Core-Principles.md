# The 5 Core APA Ethical Principles

**Source:** American Psychological Association (APA) Ethical Principles of Psychologists and Code of Conduct (2017)  
**Status:** Active standard; 2025 draft expands to 8 principles  
**Application:** All psychology-related practice, research, teaching, and technology

---

## 1. Beneficence & Nonmaleficence

### Core Intent
Psychologists **strive to benefit** those with whom they work and **take care to do no harm.**

### Application to Psychology Apps

✅ **DO:**
- Design features that demonstrably improve mental health outcomes
- Include safety rails (crisis resources, escalation to human therapist)
- Test features for unintended psychological harms
- Provide genuine value, not manipulation disguised as help
- Exit gracefully when app reaches limits of competence

❌ **DON'T:**
- Use persuasion techniques to increase engagement at the expense of user wellbeing
- Withhold crisis resources to "keep users in the app"
- Make unsupported claims about therapeutic efficacy
- Continue providing service when user needs human care
- Exploit psychological vulnerabilities (fear, loneliness, shame) for engagement

### Key Questions
1. **Does this genuinely help this person?** (Beneficence)
2. **Could this cause psychological harm?** (Nonmaleficence)
3. **Have we tested for unintended consequences?**
4. **When should we refer to a human therapist?**

### Red Flag Indicators
- [ ] Using dark patterns (notifications timed to exploit FOMO)
- [ ] Habit-formation loops without therapeutic justification
- [ ] Encouraging isolation from human relationships
- [ ] Promising cures/fixes without evidence
- [ ] Continuing service despite clear need for human intervention

## 2. Fidelity & Responsibility

### Core Intent
Psychologists **establish relationships of trust,** accept professional responsibility, and uphold standards of conduct.

### Application to Psychology Apps

✅ **DO:**
- Be transparent about what your app is and isn't
- Clearly state: "This is NOT a substitute for therapy"
- Own mistakes quickly and transparently
- Have clear responsibility chains (who fixes bugs? who responds to harm?)
- Build features explicitly designed to support therapy, not replace it
- Maintain professional boundaries even in automated interactions

❌ **DON'T:**
- Blur the line between app and therapist (e.g., "Your AI therapist")
- Disappear when something goes wrong (unresponsive support)
- Hide limitations in fine print
- Pretend to be a licensed therapist
- Shift responsibility ("it's the AI's fault")
- Create false intimacy to drive engagement

### Key Mechanisms
- **Transparency statement:** Clear disclosure on app homepage/signup
- **Error handling:** When something goes wrong, acknowledge it directly
- **Escalation protocol:** Clear path to human support 24/7
- **Terms & liability:** Honest about what app will/won't do

### Red Flag Indicators
- [ ] Marketing language suggests app = therapy
- [ ] No clear statement of limitations
- [ ] Support requests go unanswered for >24h
- [ ] Terms of service hide real risks
- [ ] App continues despite clear harm indicators
- [ ] No escalation path for crisis situations

## 3. Integrity

### Core Intent
Psychologists **promote accuracy, honesty, and truthfulness;** avoid deception and misrepresentation.

### Application to Psychology Apps

✅ **DO:**
- Be honest about how the app works (especially AI/ML components)
- Clearly state: "This app uses machine learning"
- Disclose data usage honestly (what gets saved? shared? analyzed?)
- Report research findings accurately (no p-hacking, cherry-picked studies)
- Don't exaggerate the science
- Acknowledge what we don't know
- Be transparent about conflicts of interest

❌ **DON'T:**
- Claim peer-reviewed validation that doesn't exist
- Cherry-pick studies to support marketing
- Hide how AI makes decisions (unexplainable recommendations)
- Use biased training data without disclosure
- Misrepresent research findings
- Create fake user testimonials
- Overstate scientific certainty

### Key Questions
1. **Can I defend every claim with evidence?**
2. **Am I being transparent about limitations of the science?**
3. **Does the user understand how this technology works?**
4. **Are we sharing data in ways we've disclosed?**

### Red Flag Indicators
- [ ] Marketing claims exceed research support
- [ ] No citations for therapeutic effectiveness
- [ ] AI model behavior is unexplainable or opaque
- [ ] Data practices differ from privacy policy
- [ ] Studies cherry-picked to support predetermined conclusion
- [ ] "Proprietary algorithm" used as excuse to hide methodology

## 4. Justice

### Core Intent
All persons are entitled to **equitable access and equal quality** in psychological services. Psychologists exercise reasonable judgment to avoid unjust practices based on bias, incompetence, or limitations.

### Application to Psychology Apps

✅ **DO:**
- Test app across diverse populations (race, ethnicity, gender, disability, age, SES)
- Ensure accessibility (WCAG compliance, multiple languages, cultural adaptation)
- Price equitably (low-cost or free tiers for underserved populations)
- Audit for bias in AI/recommendations
- Recruit diverse beta testers and researchers
- Address socioeconomic barriers to access
- Monitor for disparate impact across groups

❌ **DON'T:**
- Build for "average user" and ignore outliers
- Use training data that's predominantly one demographic
- Assume one language/culture fit for all
- Price out low-income users
- Ignore disability accessibility
- Deploy without testing across demographic groups
- Ignore cultural mental health differences

### Equity Checklist
- [ ] App tested with diverse user groups (minimum: race, gender, age, disability status, language)
- [ ] WCAG 2.1 AA accessibility compliance (color contrast, screen reader, keyboard nav)
- [ ] At least 2 languages supported (or plan for it)
- [ ] Pricing model includes free/low-cost access
- [ ] AI/ML models audited for bias
- [ ] Documentation available in plain language, not jargon
- [ ] Cultural norms/mental health models reflected (not just Western individualism)

### Red Flag Indicators
- [ ] App tested only with college-educated, English-speaking demographic
- [ ] No accessibility features
- [ ] Bias observed in AI recommendations by demographic group
- [ ] Pricing inaccessible to low-income users
- [ ] Overlooking cultural differences in mental health understanding
- [ ] No diversity in development/research team

## 5. Respect for People's Rights & Dignity

### Core Intent
Psychologists **respect autonomy, privacy, dignity, and cultural context.** They recognize individual differences and protect people from discrimination.

### Application to Psychology Apps

✅ **DO:**
- Get genuine informed consent (explain what you'll do with data)
- Respect user autonomy (don't manipulate choices)
- Protect privacy fiercely (encrypt data, limit access, clear retention policies)
- Allow users to delete their data
- Respect cultural/religious values (don't assume one worldview)
- Recognize diverse mental health needs (neurodivergence, trauma, LGBTQ+, etc.)
- Obtain explicit consent for any research/data sharing
- Allow users to withdraw from the app without penalty

❌ **DON'T:**
- Use dark patterns ("unsubscribe hidden in settings")
- Track/sell user data without explicit consent
- Pressure users to share more than they want
- Assume one mental health model fits all (e.g., Western individualism vs. collectivism)
- Invalidate or pathologize cultural/spiritual beliefs
- Discriminate against marginalized groups
- Use defaults that expose privacy
- Collect data beyond what's needed

### Consent Framework
**Real informed consent means:**
1. User understands WHAT data you collect
2. User understands WHY you need it
3. User understands HOW it's used/shared
4. User understands RISKS
5. User can say NO without penalty
6. User can WITHDRAW anytime

### Red Flag Indicators
- [ ] Privacy policy uses legal jargon; user doesn't understand
- [ ] Data deletion requires manual intervention
- [ ] Dark patterns (buried unsubscribe, auto-renewal, etc.)
- [ ] User research data shared without explicit consent
- [ ] Notifications designed to manipulate behavior
- [ ] App shames/judges user values/beliefs
- [ ] Assumes one mental health model for all demographics

## Integration Framework: When Principles Conflict

Real-world design often creates tension between principles. Use this framework:

### Example: Beneficence vs. Autonomy
**Scenario:** App learns user is suicidal but in denial; should app force hospitalization recommendation?

**Resolution:**
1. **Beneficence:** Protecting life is the highest good → recommend crisis resources
2. **Autonomy:** User can reject recommendation → respect their choice
3. **Synthesis:** Provide clear crisis info, explain risk, allow user to proceed with caution. Document effort.

### Example: Fidelity vs. Justice
**Scenario:** Translating app to new language is expensive; restrict to wealthy countries only?

**Resolution:**
1. **Fidelity:** Be honest about limitations
2. **Justice:** Ensure equitable access
3. **Synthesis:** Release English/Spanish beta widely, crowdfund/partner for expansion to underserved languages

## Translation to Technical Design

### Beneficence/Nonmaleficence
- Code review: "Does this cause harm?"
- Testing: Adverse event monitoring (what % of users report worse mood?)
- Design: Crisis rails, escalation paths, evidence-based features

### Fidelity/Responsibility
- Docs: Clear disclaimers visible on signup
- Engineering: Incident response SLA <1h
- Support: 24/7 escalation path

### Integrity
- Data: Audit AI model decisions for bias/transparency
- Marketing: Claims require citations or disclaimer
- Research: Pre-register studies, report null findings

### Justice
- QA: Test across demographics before launch
- Access: Pricing tiers, language support, WCAG compliance
- Hiring: Diverse team reflects diverse users

### Respect
- Consent: Plain-language privacy explanation
- UX: One-click data deletion
- Design: No dark patterns, no shame/judgment

## Next Steps: The 2025 Draft

The APA is expanding from 5 to **8 principles:**

1-5: (Current principles, refined)
6. **Human & Civil Rights** — Protect vulnerable populations, oppose discrimination
7. **Scientific Mindedness** — Use evidence, embrace uncertainty, update views
8. **Environmental Stewardship** — Consider impact on world/community wellbeing

These are expected to be finalized 2025-2026. Apps should already be considering them.
